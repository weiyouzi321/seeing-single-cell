## 设计经验总结（Ch1-4）

### 成功的设计模式
1. **四步递进法**：生物直觉 → 数据可视化 → 算法交互 → 结果解读
2. **「装水的袋子」类比**：复杂概念用日常直觉锚定（Ch2 质控）
3. **可操控的按钮/滑块**：让用户「用手感受数学」（归一化按钮、投影角度滑块）
4. **Before/After 并排对比**：每个变换的效果一目了然（归一化、ScaleData、PCA步骤）
5. **生物学驱动的数学解释**：不说「方差最大化」，先说「哪些基因能把细胞分开」

### 技术模式
- 数据：100细胞 × 52基因，6种 PBMC 类型（CD4 T, CD8 T, B, NK, Monocyte, DC）
- 管线：raw → normalize(log1p) → HVG(20个) → scale(z-score) → PCA → 聚类 → 降维可视化 → DEG
- 可视化：p5.js 画布 + useMemo 计算 + KaTeX 公式
- 每个 viz 组件接收 `(data, geneNames, cellTypes, lang, activeStep)` 参数

### 项目理念
> 交互探索 / 观察与理解 / 建立直觉
> "理解为什么这样做，而不仅仅是怎么做"

## Chapter 5: KNN 图聚类（Graph-based Clustering）

### 生物场景
> PCA 把 20 维压缩到了 10 个 PC。现在的问题是：这些 PC 空间中的细胞，能不能自动分成几个「群」？就像在一个操场上，你一眼就能看出哪些人在一起玩。

### 四步设计

**Step 1: 什么是「距离」？**
- 核心问题：两个细胞有多「像」？
- 交互：选两个细胞，展示它们的基因表达向量，计算欧氏距离和余弦相似度
- 可视化：两个柱状图并排（细胞 A vs 细胞 B 的各基因表达），距离值实时显示
- 数学：欧氏距离 d = √Σ(aᵢ-bᵢ)²，余弦相似度 cos(θ) = A·B / (|A||B|)
- 生物类比：两个人的「口味 profile」——如果都喜欢辣和甜，距离就近

**Step 2: K 近邻图（KNN Graph）**
- 核心概念：每个细胞找 K 个最近的邻居，连成一张图
- 交互：滑块调整 K 值（1-20），实时看到图的连接变化
- 可视化：PC1 vs PC2 散点图 + 细胞间的连线（KNN 边）
- 关键洞察：K 太小 → 碎片化；K 太大 → 所有细胞都连在一起
- 引入 Louvain 的动机：有了邻居关系图，怎么找到「社区」？

**Step 3: Louvain 社区发现**
- 核心思想：模块度（Modularity）最大化
- 可视化：动画展示 Louvain 迭代过程
  - 初始：每个细胞是一个独立社区（不同颜色）
  - 每轮：尝试合并相邻社区，如果模块度增加就合并
  - 最终：稳定在几个大社区
- 交互：「播放/暂停/单步」按钮控制迭代
- 数学：Q = (1/2m) Σ[Aᵢⱼ - kᵢkⱼ/2m] δ(cᵢ,cⱼ)
- 生物类比：微信群——自动找到聊得最频繁的一群人

**Step 4: 聚类结果评估**
- 可视化：聚类结果 vs 真实标签的 confusion matrix（热图）
- 交互：调整 K 值和分辨率参数，观察聚类粒度变化
- 指标：ARI（调整兰德指数）、NMI（标准化互信息）
- 「如果不过滤」的教训延伸：K 选错会怎样？

### 流程图
```
PCA (20 HVGs × 100 cells, 10 PCs)
  → 计算细胞间距离 (欧氏/余弦)
  → 构建 KNN 图 (K=10)
  → Louvain 社区发现
  → 细胞类型标签 (Cluster 1-6)
```

## Chapter 6: 降维可视化（t-SNE & UMAP）

### 生物场景
> PCA 是线性的——它找到最佳的「直线投影」。但生物学不是线性的。就像地球表面不能完美摊平成一张地图一样，高维的细胞空间可能有弯曲的结构。t-SNE 和 UMAP 是两种「非线性地图绘制」方法。

### 四步设计

**Step 1: 线性 vs 非线性降维**
- 核心问题：为什么 PCA 不够？
- 交互：展示一个「瑞士卷」数据集
  - PCA 展开 → 两个类混在一起
  - t-SNE/UMAP 展开 → 两个类清晰分开
- 生物类比：把一个皱巴巴的纸团展开——线性方法像从上面压扁，非线性方法像小心地展开
- 可视化：三个并排散点图（PCA / t-SNE / UMAP）对比同一个数据集

**Step 2: t-SNE 的工作原理**
- 核心概念：保持局部结构
- 数学分步讲解：
  - ① 高维空间：计算点对相似度（高斯核），P(j|i) = exp(-||xᵢ-xⱼ||²/2σ²) / Σ
  - ② 低维空间：初始化点位置，计算 t 分布相似度 Q(j|i)
  - ③ 最小化 KL 散度：把 P 和 Q 拉近
- 交互：
  - 「困惑度（Perplexity）」滑块：5-50，观察聚类数量和紧密度变化
  - 「学习率」滑块：观察收敛速度
  - 「迭代次数」播放按钮：看点从随机分布逐渐聚成团的过程动画
- 关键洞察：困惑度 ≈ 有效邻居数，类似 KNN 的 K

**Step 3: UMAP 的工作原理**
- 核心概念：同时保持局部和全局结构
- 与 t-SNE 对比：
  - t-SNE：只关注局部（近邻保持），全局结构可能失真
  - UMAP：局部+全局都保持，计算更快
- 数学直觉（简化）：
  - ① 构建高维模糊单纯复形（fuzzy simplicial complex）
  - ② 在低维空间中重建类似的拓扑结构
  - ③ 交叉熵损失函数优化
- 交互：
  - 「n_neighbors」滑块：控制局部 vs 全局的权衡
  - 「min_dist」滑块：控制聚类的紧密度
  - 迭代动画
- 生物类比：UMAP 像一张更好的地图——既有街道细节，又有城市间的距离关系

**Step 4: 结果比较与陷阱**
- 并排对比：同一数据的 PCA / t-SNE / UMAP
- 常见陷阱：
  - t-SNE 不同随机种子 → 不同的图（结果不唯一）
  - 距离不代表真实的高维距离（簇间距离无意义）
  - Perplexity 选择不当 → 过度分割或过度合并
- 交互：多次点击「重新运行」看 t-SNE 的不稳定性
- 何时用什么：PCA（快速预览）→ t-SNE（发表级图）→ UMAP（大数据集）

### 流程图
```
标准化数据 (20 HVGs)
  → PCA (降到 10-50 PCs, 去噪声)
  → t-SNE / UMAP (降到 2D)
  → 可视化细胞类型分布
```

## Chapter 7: 差异表达基因分析（DEG）

### 生物场景
> 我们已经把细胞分成了 6 个群。现在要问：群 1（T 细胞）和群 2（B 细胞）之间，哪些基因的表达量明显不同？这些基因就是差异表达基因（DEG），它们是理解细胞身份的钥匙。

### 四步设计

**Step 1: 什么是「差异表达」？**
- 核心问题：怎么判断一个基因在两组细胞之间「真的不同」？
- 交互：选一个基因（如 CD3D），展示它在 Cluster 1（T 细胞）和 Cluster 2（B 细胞）中的表达分布
  - 小提琴图并排
  - 箱线图叠加
  - 均值差异和 fold change 实时计算
- 生物类比：两个班的考试成绩——平均分差 5 分是「显著」吗？要看标准差和人数
- 数学：log₂FC = log₂(mean_B) - log₂(mean_A)

**Step 2: 统计检验入门**
- 核心概念：差异是不是碰巧出现的？
- 交互：模拟实验
  - 从同一个正态分布中随机抽两组样本
  - 计算 t 统计量和 p 值
  - 重复 1000 次，画 p 值分布直方图
  - 展示：如果真的没有差异，p < 0.05 的概率就是 5%（假阳性率）
- 数学：t = (mean₁ - mean₂) / (SE₁² + SE₂²)^0.5
- 「多重检验」问题引入：10000 个基因都做 t 检验 → 500 个假阳性！

**Step 3: Wilcoxon 秩和检验（单细胞标准方法）**
- 为什么不用 t 检验？单细胞数据不满足正态性假设
- Wilcoxon 检验的直觉：
  - 把两组数据合并，按大小排名
  - 如果一组的排名明显偏高 → 差异显著
  - 不依赖分布假设
- 交互：
  - 选一个基因
  - 展示两组细胞的排名分布
  - 计算 U 统计量
  - 显示 p 值和 log₂FC
- 可视化：火山图（Volcano Plot）
  - X 轴：log₂FC（差异大小）
  - Y 轴：-log₁₀(p-value)（显著性）
  - 红色点：显著上调基因
  - 蓝色点：显著下调基因
  - 交互：点击一个点显示基因名和详情

**Step 4: 多重检验校正与结果解读**
- 核心问题：10000 个基因同时检验，怎么控制假阳性？
- 交互：Benjamini-Hochberg 校正过程动画
  - ① 按 p 值排序
  - ② 计算 adjusted p = p × n / rank
  - ③ 画出原始 p 值 vs adjusted p 值
  - ④ 在 FDR = 0.05 处画线
- 最终结果展示：
  - 火山图（校正后的 adjusted p）
  - 热图：top 20 DEG 在各 cluster 中的表达
  - 基因功能注释（CD3→T cell, CD19→B cell, etc.）
- 生物解读：这些 DEG 告诉我们每个 cluster 的「身份」

### 流程图
```
聚类结果 (6 clusters)
  → 每对 cluster 做 DEG 分析
  → Wilcoxon 秩和检验
  → 多重检验校正 (BH/FDR)
  → 火山图 + 热图
  → 标记基因列表
```

## Chapter 8: SCTransform（高级归一化）

### 生物场景
> 之前我们用的「除文库大小 + log1p」归一化是最简单的方法。但它有个问题：它假设所有技术噪声都可以用文库大小来解释。实际上，不同基因有不同的技术噪声水平。SCTransform 用更精细的统计模型来处理这个问题。

### 四步设计

**Step 1: 简单归一化的局限性**
- 回顾：Ch3 的 log1p 归一化做了什么
- 问题暴露：
  - 可视化：归一化后，某些基因仍有「漏斗形」的均值-方差关系
  - 低表达基因的方差被高估
  - 这会干扰 HVG 选择
- 交互：对比 log1p 和 SCTransform 的均值-方差图
- 生物类比：简单归一化像用同一个尺子量所有人——不管高矮胖瘦

**Step 2: 负二项回归模型**
- 核心思想：为每个基因建立一个「表达量 ~ 文库大小」的回归模型
- 数学：Y ~ NB(μ, θ)，其中 log(μ) = β₀ + β₁·log(nCount)
- 交互：
  - 选一个基因
  - 展示该基因在所有细胞中的表达量 vs 文库大小的散点图
  - 叠加负二项回归曲线
  - 残差 = 归一化后的值
- 关键洞察：残差已经去除了文库大小的影响

**Step 3: Pearson 残差**
- 公式：z = (observed - expected) / √(expected + expected²/θ)
- 交互：
  - 对比 log1p 和 Pearson 残差的分布
  - 展示残差的均值 ≈ 0，方差 ≈ 1
  - 观察残差不再有均值-方差依赖
- 可视化：热图对比（log1p 矩阵 vs SCT 矩阵）

**Step 4: SCTransform 的效果**
- 对比下游分析：
  - PCA：SCT → 更清晰的 cluster 分离
  - HVG 选择：不再受低表达基因噪声干扰
  - 批次效应：SCT 部分缓解技术变异
- 何时用 SCT vs log1p：
  - SCT：更准确，适合复杂数据集，计算量更大
  - log1p：简单快速，适合数据质量好的情况
- 交互：同一数据集的 log1p vs SCT → PCA → UMAP 全流程对比

### 流程图
```
Raw counts
  → 负二项回归 (per gene)
  → Pearson 残差
  → 剪切到 [-30, 30] 范围
  → 正则化回归 (regularized regression)
  → SCT 标准化矩阵
  → PCA → 聚类
```

## Chapter 9: 去批次效应（Batch Correction）

### 生物场景
> 你有来自 3 个不同患者的样本。每个患者的细胞在 PC 空间中聚成一团——这是因为真的不同，还是因为测序批次不同？批次效应是单细胞分析中最头疼的问题之一。CCA 和 Harmony 是两种常用的去批次方法。

### 四步设计

**Step 1: 什么是批次效应？**
- 核心概念：技术变异 vs 生物变异
- 交互：模拟 3 个批次的数据
  - 批次 1：CD4 T 偏高，B 偏低
  - 批次 2：整体偏低
  - 批次 3：NK 偏高
- 可视化：PCA 散点图，按批次着色 vs 按细胞类型着色
  - 如果按批次着色能分出 3 个簇 → 有批次效应
  - 如果按细胞类型着色能分出 6 个簇 → 生物信号还在
- 生物类比：三个不同品牌的体重秤——同一个人称出来不一样，这就是批次效应

**Step 2: CCA（典型相关分析）**
- 核心思想：找到两个数据集共享的「共同空间」
- 数学直觉：
  - PCA：一个数据集内找最大方差方向
  - CCA：两个数据集之间找最大相关方向
- 交互：
  - 展示两个批次的 PC 空间
  - CCA 找到对齐方向
  - 动画：两个批次的点在 CCA 空间中逐渐靠近
- Seurat 的 CCA 流程：
  - ① 各自做 PCA
  - ② CCA 找到锚点（anchors）
  - ③ 用锚点对齐数据
- 局限性：只处理两个批次，多批次需要迭代

**Step 3: Harmony**
- 核心思想：在 PCA 空间中迭代调整每个细胞的位置，让不同批次的细胞混合
- 算法直觉（简化版）：
  - ① 所有细胞的 PCA 坐标
  - ② 对每个 cluster，计算每个批次的中心
  - ③ 把每个细胞向「全局中心」拉近（去除批次偏移）
  - ④ 重复直到混合
- 交互：动画展示迭代过程
  - 初始：3 个批次的细胞各自成团
  - 每轮：细胞逐渐向全局中心移动
  - 最终：不同批次的同类型细胞混合在一起
- 与 CCA 对比：
  - CCA：线性变换，适合两个批次
  - Harmony：非线性调整，适合多批次，更快更灵活
- 可视化：Before/After PCA 散点图（按批次着色）

**Step 4: 去批次效果评估**
- 评估指标：
  - 混合度：同类型细胞是否跨批次混合？
  - 生物保留度：不同类型的细胞是否仍然分开？
- 交互：
  - 切换 Before/After 视图
  - 按批次着色 / 按类型着色对比
  - 计算 LISI（Local Inverse Simpson's Index）得分
- 常见陷阱：
  - 过度校正：把真的生物差异也消除了
  - 校正不足：批次效应还在
  - 选择什么作为「批次」变量很重要
- 何时用什么：
  - CCA：经典方法，适合两个批次，Seurat 默认
  - Harmony：现代首选，多批次高效，保留生物信号

### 流程图
```
多个批次的 PCA 坐标
  → 检测批次效应（按批次着色的 PCA）
  → CCA 锚点对齐 / Harmony 迭代校正
  → 校正后的 PCA 坐标
  → 重新聚类
  → 评估混合度和生物保留度
```

## 开发优先级建议
1. **Ch5 (KNN聚类)** — 最紧密衔接 Ch4 PCA，逻辑上最自然
2. **Ch6 (t-SNE/UMAP)** — Ch5 之后自然延伸，结果可视化
3. **Ch7 (DEG)** — 需要聚类结果，顺序上排第三
4. **Ch8 (SCTransform)** — 回顾 Ch3 的归一化，属于进阶内容
5. **Ch9 (批次校正)** — 最后讲，因为需要理解前面所有步骤
